## 1. 目录结构规范

项目必须保持清晰的模块化结构，所有新增文件需归属于以下指定目录：

* **`components/` (组件层)**: 存放跨页面复用的公共 UI 组件。
* **`pages/` (页面层)**: 存放具体的业务页面组件及页面级逻辑。
* **`utils/` (工具层)**: 存放通用的辅助函数、格式化工具及业务逻辑抽离。
* **`styles/` (样式层)**: 存放全局样式、主题配置（如 Less/ConfigProvider）及公共变量。


## 2. 组件开发准则

在使用 **React** 和 **Ant Design (antd)** 构建组件时，必须遵循以下要求：

### 2.1 命名规范

* **组件名称**: 必须使用 **大驼峰命名法 (PascalCase)**。例如：`UserProfileCard`。
* **内部变量/函数**: 采用 **小驼峰命名法 (camelCase)**。

### 2.2 注释与文档说明

每个组件的顶部必须包含标准的 JSDoc 注释块，详细说明其职责：

| 标注 | 描述 |
| --- | --- |
| **@description** | 简述组件的功能及应用场景 |
| **@param** | 定义属性 (Props) 的名称、类型及用途 |
| **@return** | 说明组件渲染的内容或返回的 JSX |
| **@example** | (可选) 提供组件调用的代码示例 |


## 3. UI/UX 约束

* **组件库**: 优先使用 **Ant Design** 官方组件。
* **布局**: 优先使用 `Row/Col` 或 `Space` 组件进行排版，避免使用原生 CSS 的复杂硬编码。
